Git & Termux Command Guide

This guide explains all commands in sequence, with breakdowns, tips, and copy buttons. Color-coded explanations are used for memory psychology.

1. Navigate to Folder

cd /storage/emulated/0/YourFolder

cd: Change directory command in Linux/Termux.

/storage/emulated/0/YourFolder: Path to your folder in Android storage.

Tips: Use relative or absolute paths. `cd ..` moves one folder up.

2. Initialize Git Repo

git init

git: Version control system (tracks changes, allows collaboration).

init: Initializes a new Git repository in the current folder.

First step to make a folder a Git repository.

3. Add Remote Repository

git remote add origin https://github.com/YourUsername/YourRepo.git

git: Git tool.

remote: Manage remote repositories.

add: Add a new remote repository.

origin: Default name for the remote repository.

URL: Your GitHub repo link.

Use personal access token if GitHub asks for password.

4. Add Files

git add .

git: Git tool.

add: Stage changes for commit.

. Current folder and all contents.

Stage before committing.

5. Commit Changes

git commit -m \"Initial commit\"

git: Git tool.

commit: Record staged changes.

-m \"Initial commit\": Commit message describing changes.

Use descriptive messages for clarity.

6. Push to GitHub

git push origin main

push: Send commits to remote repository.

origin: Remote repo name.

main: Branch to push to.

Force push only if rewriting history.

7. Pull from GitHub

git pull origin main

pull: Fetches and merges remote changes.

origin: Remote repository.

main: Branch to pull from.

Always pull before making new changes.

8. List Files Recursively

ls -R

ls: List files.

-R: Recursive listing of all subfolders.

Use `ls -a` to include hidden files.

9. View File Content

cat index.html

cat: Display file content.

index.html: File to view.

Use `less` to scroll, `nano` to edit.

10. Copy File Content to Clipboard

cat index.html | termux-clipboard-set

| Pipe output to next command.

termux-clipboard-set: Copies content to Android clipboard.

Requires Termux API.

11. Move Folder / Files

mv project/* .

mv: Move files/folders.

project/*: All files in project folder.

. Current folder (destination).

Always move inside repo to preserve Git tracking.

12. Delete File

rm filename

rm: Delete file permanently.

filename: File name.

Git tracks deletion after `git add .`

13. Delete Folder

rm -r foldername

-r: Recursive deletion of folder and contents.

Permanent deletion; be careful.

14. Reset Last Commit

git reset --soft HEAD~1

--soft: Keep changes but remove commit.

HEAD~1: Last commit.

Use --hard to discard changes completely. Force push required if already on GitHub.

15. Delete All Commit History

git checkout --orphan temp_branch && git add . && git commit -m \"Initial commit\" && git branch -D main && git branch -m main && git push origin main --force

Sequence to delete all previous commits but keep files.

Force push updates GitHub repository.
